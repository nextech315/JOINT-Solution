The Problem

기존 설계 방식의 한계

시작(Try-out) 제작과 반복 TEST에 의존하던 Joint 설계 검증, 이제는 AI 예측으로 대체할 때입니다.

🔧

반복되는 시작(Try-out) 제작

설계를 바꿀 때마다 목업을 만들고 TEST를 반복해야 합니다. Breakaway Torque, 유격 등 핵심 품질은 실제로 만들어봐야 확인할 수 있습니다.

🧩

34개 변수의 복잡한 상호작용

설계 변수 33개와 공정 변수 1개가 10개 품질 타겟에 동시에 영향을 미칩니다. 사람이 손으로 최적 조합을 찾기엔 경우의 수가 너무 많습니다.

⏳

길어지는 설계 검증 주기

도면을 변경할 때마다 시작 제작과 TEST에 수일에서 수주가 소요됩니다. 양산 일정에 맞춰 설계를 확정하기가 촉박합니다.

✅

NexTech Joint AI 솔루션

실측 데이터를 학습한 AI가 설계 변경 전에 품질 결과를 미리 예측하고, 목표 품질을 만족하는 최적 설계·공정 조건을 수 초 만에 역으로 계산합니다.

How It Works

3단계로 완성되는
AI 설계·공정 최적화

데이터를 올리는 순간부터, AI가 목표 품질을 만족하는 조건을 찾아 즉시 적용 가능한 답을 드립니다.

데이터 업로드 & AI 자동 학습

Tab 1 · 역추론 최적화

Tab 2 · 순방향 예측

설계 & 공정 변수 타겟팅 최적화

이미 정해진 설계·공정 조건의 범위를 입력하면, 그 범위 안에서 예상되는 10개 품질 타겟 값을 순방향으로 예측합니다. "설계 범위는 정해져 있는데 스펙을 맞출 수 있을지 궁금할 때" 사용합니다.

🔄

2-Way 최적화 엔진

품질 목표 → 설계 조건 역산(Tab1)과 설계 조건 → 품질 예측(Tab2)을 모두 지원합니다. 상황에 맞는 방향으로 계산합니다.

📝

LLM 기반 공정 가이드

달성 불가 타겟 분석, 스펙 이탈 분석, Feature Importance 기반 조정 가이드, 단계별 Action Plan을 자동 생성합니다.

AI Engine

타겟마다 다른
최적 알고리즘을
자동 선택

10개 품질 타겟은 서로 다른 패턴을 가집니다. 데이터 특성에 맞춰 학습 알고리즘과 최적화 알고리즘을 자동으로 조합합니다.

Linear

Linear Regression / Ridge

변수 간 선형 관계를 빠르게 파악하는 기준 모델. 공정 변수 변화가 결과에 미치는 영향을 직접적으로 계산합니다.

Tree

RandomForest Regressor

여러 결정 트리를 앙상블해 비선형 패턴에 강하고, 변수가 많고 복잡한 Joint 데이터에서도 안정적인 성능을 보입니다.

Boost

XGBoost / GradientBoosting

오차를 보완하며 학습하는 부스팅 방식. 정형 데이터 분석에서 가장 정밀한 예측 성능을 제공하며 KPI 예측 정확도의 핵심 역할을 합니다.

최적화 엔진 — 손실 함수 최소화
SLSQP

제약 조건이 있는 비선형 최적화에 특화

L-BFGS-B

메모리 효율적, 변수가 많아도 빠르게 수렴

Powell

미분 없이 최적화, 불연속 함수에도 대응

Nelder-Mead

그래디언트 없이 동작하는 대안적 탐색

각 타겟마다 손실 함수의 형태가 다르기 때문에 4종 알고리즘을 모두 실행한 뒤 가장 낮은 손실 값을 채택합니다. 손실 함수에는 달성 가능성을 반영하는 Feasibility 가중치와, 현실적인 범위를 벗어나지 않도록 하는 정규화 패널티가 포함됩니다.
💡 Target Optimization Confidence 100%란? AI 정확도가 100%라는 뜻이 아니라, 설정한 품질 목표에 대해 계산된 값이 얼마나 근접한지를 백분율로 나타낸 지표입니다. 타겟 범위가 넓고 계산 값이 범위 중심에 가까울수록 100%에 가까워지고, 범위가 좁거나 경계에 걸쳐 있을수록 낮아집니다.
